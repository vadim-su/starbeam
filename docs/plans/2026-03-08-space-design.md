# Дизайн: Космос

## Обзор

Космос — полноценное игровое пространство. Корабль — такой же мир как планета, со своей тайлмапой, строительством и механиками. Игрок перемещается между поверхностью планет и кораблём через капсулы, а между небесными телами — через консоль автопилота на корабле.

## Мир-корабль

- Адресуется как `CelestialAddress::Ship`
- Обычная тайлмапа через существующую chunk-систему и persistence
- Генерируется процедурно "пустым" генератором — воздух/пустота вместо terrain
- Начальный каркас (минимальная структура + функциональные блоки) — часть генерации
- Расширяемый размер: начинается маленьким, игрок достраивает через крафт модулей расширения
- Wrap-around выключен — новая настройка мира `wrap_x: bool` (планеты — включен, корабль — выключен)
- За пределами построенных блоков — открытый космос

### Слои

Те же foreground/background что на планетах:
- Foreground — каркас корабля, полы, стены (участвуют в коллизиях)
- Background — внутренняя отделка (декоративно, не участвует в коллизиях)
- Parallax — космический фон текущей локации

## Капсула (переход планета ↔ корабль)

- Размещаемый объект на поверхности планеты (система `object/`)
- Игрок взаимодействует с капсулой → UI "Подняться на корабль"
- Срабатывает `WarpToBody` с адресом корабля — переиспользуем существующую warp-систему
- Игрок появляется на корабле в точке спавна

### Спуск обратно

- На корабле есть шлюз (функциональный блок)
- Взаимодействие → "Спуститься на планету"
- Варп обратно на планету, игрок появляется рядом с капсулой
- Спуск доступен только если корабль на орбите этого тела

### Ограничения

- Одна капсула на планету на игрока
- Капсула запоминает координаты на планете
- Корабль помнит, на орбите какого тела находится

## Навигация и перелёт

### Консоль автопилота

- Функциональный блок, размещаемый на корабле
- Без консоли корабль не летает
- Взаимодействие → открывается Star Map UI (существующий)
- Выбор целевого тела (планета, луна, станция)
- Если достаточно топлива → запуск перелёта

### Перелёт в реальном времени

- Игрок свободно ходит по кораблю, строит, крафтит
- Длительность зависит от расстояния между телами в системе
- Параллакс-фон плавно меняется (текущая орбита → deep space → целевая орбита)
- Спуск через шлюз недоступен во время полёта
- По прибытии — уведомление, корабль выходит на орбиту цели

### Топливо

- Ресурс, хранится в топливном баке (функциональный блок на корабле)
- Расход пропорционален расстоянию
- Добывается на планетах / крафтится
- Без топлива — никуда не летишь

## Открытый космос (EVA)

За бортом корабля — открытый космос. Игрок может выйти за пределы блоков.

### Физика

- Гравитация 0 — игрок плавает
- Перемещение через джетпак (встроен в скафандр) — WASD даёт импульс
- Инерция — игрок продолжает двигаться, постепенно замедляясь (небольшое трение для играбельности)

### Кислород

- Таймер, расходуется за бортом
- Внутри корабля (герметичная зона) — восстанавливается
- Кончился → урон
- Запас увеличивается через экипировку

## Герметичность (flood-fill)

Определяет, где "внутри корабля" (атмосфера), а где "космос".

### Алгоритм

- Flood-fill от краёв мира: всё пространство, достижимое от края без прохода через foreground-блоки = космос
- Остальное = герметичная зона (атмосфера)
- Результат кешируется, пересчёт только при изменении блоков

### Эффекты

- Атмосфера: нормальная гравитация + кислород не расходуется
- Космос: невесомость + расход кислорода
- Пробоина в корпусе = разгерметизация (геймплейная механика)

## Параллакс — космические биомы

Вместо отдельной системы локаций — переиспользуем биомы.

### Космические биомы

- `orbit_garden` — орбита garden-планеты (видна поверхность, атмосферное свечение)
- `orbit_barren` — орбита barren-планеты
- `deep_space` — полёт между телами (только звёзды)
- `orbit_station` — орбита станции

### Реализация

- Мир-корабль имеет глобальный биом (один на весь мир, не зонированный)
- Новый флаг в конфиге мира: `global_biome: true`
- Биом меняется при смене локации корабля
- Плавная смена через существующий parallax transition (`parallax/transition.rs`)

## Функциональные блоки корабля (минимум)

| Блок | Назначение |
|------|-----------|
| Консоль автопилота | Навигация, открывает Star Map |
| Топливный бак | Хранение топлива для перелётов |
| Шлюз | Спуск на планету / подъём с планеты |

## Архитектурные решения

- **Подход A:** корабль как специальный мир в существующей системе — максимальное переиспользование кода (тайлмапа, чанки, warp, persistence)
- Wrap-around — настройка мира (`wrap_x: bool`), не хардкод
- Биомы для космических локаций — без новой системы, переиспользуем параллакс + transition
- Глобальный биом — расширение biome_map для миров без зонирования
