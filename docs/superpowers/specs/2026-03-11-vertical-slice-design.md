# Vertical Slice Design — одна планета с полным набором взаимодействий

## Цель

MVP в формате vertical slice: одна планета, на которой игрок может сражаться с врагами, торговать с NPC, прогрессировать через инструменты и справляться с опасностями окружения.

---

## 1. Боевая система

### Здоровье и урон
- Компонент `Health { current: f32, max: f32 }` — универсальный для игрока и мобов.
- Компонент `DamageEvent { amount: f32, knockback: Vec2, source: Entity }` — событие урона.
- После получения урона — ~0.5с неуязвимости (мигание спрайта через `InvincibilityTimer`).
- Смерть игрока = респаун на корабле.

### Ближний бой
- Хитбокс появляется перед игроком на время атаки, привязан к arm aiming системе.
- Урон и нокбэк зависят от `ToolStats` предмета в руке.

### Дальний бой
- Снаряд — entity с `Velocity`, `DamageSource`, коллизия с тайлами и мобами.
- Расход боеприпасов из инвентаря.

---

## 2. Враги

3 типа для vertical slice, AI реализован на `statig` (иерархические стейт-машины).

### Типы
- **Слайм** — патрулирует платформу, агрессия по радиусу ~8 тайлов, прыгает к игроку, контактный урон.
- **Стрелок** — медленно ходит, стреляет снарядами по радиусу ~12 тайлов.
- **Летающий** — летает в зоне, агрессия по радиусу, движется к игроку по воздуху.

### AI стейт-машина (statig)
- `Idle` — патрулирование / стояние на месте.
- `Alert` — игрок в радиусе обнаружения.
- `Chase` / `Attack` — движение к игроку + атака.
- `Return` — игрок далеко, возврат на исходную позицию.

### Спрайты
- Генерация через PixelLab.
- Анимации: idle, move, attack, death.

---

## 3. Разрушение блоков

- Каждый тип блока имеет `hardness: f32` — базовое HP.
- Инструмент имеет `mining_power: f32` — урон по блоку в секунду.
- Любая кирка может копать любой блок, но скорость зависит от соотношения `mining_power / hardness`.
- Визуальные стадии разрушения (3-4 стадии overlay с трещинами).
- Если игрок прекратил копать — блок регенерирует HP через таймер.
- При полном разрушении — дроп предмета.

---

## 4. Прогрессия инструментов

- 2-3 уровня: камень → железо → advanced.
- Основная прогрессия через `mining_power` (кирки) и `damage` (оружие).
- Архитектура `ToolStats { mining_power: f32, damage: f32, knockback: f32, attack_speed: f32, ... }` — расширяема для будущих свойств (радиус добычи, автосбор и т.д.).
- Рецепты крафта привязаны к уровню руды (камень → железная руда → редкая руда).

---

## 5. NPC-торговец

- Entity со спрайтом, стоит на месте (без AI).
- Компонент `Trader { offers: Vec<TradeOffer> }`.
- `TradeOffer { cost: Vec<(ItemId, u32)>, result: (ItemId, u32) }`.
- Данные загружаются из RON-файла (data-driven).
- Взаимодействие: подойти → подсветка (existing interactables) → клавиша → UI-панель торговли.
- UI: список предложений, иконки cost/result, кнопка "Обменять" (серая если не хватает ресурсов).
- Для vertical slice — заскриптованная точка спавна на планете.

---

## 6. Опасности окружения

### Урон от падения
- Отслеживание вертикальной скорости при приземлении.
- Порог: скорость > N → урон пропорционален превышению.
- Визуальный эффект (частицы).

### Урон от лавы
- Компонент `DamageOnSubmersion { dps: f32 }` на жидкостях.
- Лава наносит урон в секунду (система submersion уже есть).

### Удушье
- Кислородная система уже реализована.
- При 0 кислорода — урон в секунду по здоровью.

### Враждебная среда
- Глубокие пещеры — темнота + спавн мобов.
- Поверхность ночью — повышенный множитель спавна.

---

## 7. Спавн мобов

- `SpawnZone` — компонент на чанке/регионе: какие мобы, плотность, лимит.
- Спавн за пределами экрана, в загруженных чанках.
- Респавн через таймер после убийства.
- Ночной множитель спавна.
- В пещерах — свои типы мобов.

### Лут
- `LootTable { drops: Vec<(ItemId, u32, f32)> }` — предмет, количество, шанс.
- Data-driven из RON.

---

## 8. Ресурсная прогрессия на планете

- **Поверхность:** дерево, камень, базовые руды.
- **Средние пещеры:** железо, кристаллы.
- **Глубокие пещеры:** редкие руды (для лучших инструментов) + сильные мобы.

---

## Технические решения

- **AI:** `statig` крейт для иерархических стейт-машин врагов.
- **Спрайты:** PixelLab для генерации пиксель-арт спрайтов врагов и NPC.
- **Data-driven:** все определения (мобы, лут, торговля, инструменты) в RON-файлах.
- **Архитектура:** расширяемые компоненты (`ToolStats`, `LootTable`) для будущего роста.
